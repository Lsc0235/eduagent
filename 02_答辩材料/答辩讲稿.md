# 智学通 A3 答辩讲稿

## 1. 开场 30 秒

各位评委老师好，我们的作品是 **智学通 EduAgent**，对应中国软件杯 A3 赛题“基于大模型的个性化资源生成与学习多智能体系统”。

我们选择“人工智能导论”作为示范课程，目标不是简单做一个聊天机器人，而是做一个能完成 **画像采集、资源生成、路径推荐、在线测评、错因补强、效果评估** 的学习闭环系统。

## 2. 痛点和方案 1 分钟

高等教育里的个性化学习有三个难点：

1. 学生差异很大，但传统平台通常用固定资源推荐。
2. 大模型能生成内容，但缺少画像、知识图谱和测评闭环约束。
3. 学习系统常常只展示结果，不能解释“为什么这样推荐”。

智学通的解决方案是：

- 用自然语言对话构建 7 维动态学生画像。
- 用 Planner 编排多个专职 Agent 协同生成资源和建议。
- 用课程知识图谱组织知识点先修关系。
- 用测评和补强记录验证学习效果。

## 3. 系统架构 1 分钟

系统采用 React + FastAPI 前后端分离架构。后端包括 API 层、Agent 层、知识库层、数据库层和大模型封装层。

核心是多智能体编排：

- Planner 负责识别意图和拆解任务。
- Profiler 负责画像抽取和更新。
- Resource Agent 负责生成课程讲解、思维导图、练习题、拓展阅读和代码案例。
- Quiz Agent 负责出题和批改。
- Tutor Agent 负责答疑。
- Evaluator 负责学习效果评估和下一步建议。

复杂任务会进入 Planner 到多 Agent 并行执行再到 Aggregator 聚合的流程。

## 4. 演示路线 4 分钟

### 第一步：竞赛驾驶舱

打开 `/dashboard`。

这里展示赛题命中度、画像覆盖、资源数量、知识图谱节点，以及答辩演示路径。点击“准备演示数据”后，系统会准备完整闭环证据。

重点讲：

- 画像覆盖 7/7。
- 资源覆盖 5 类。
- 知识图谱有 16 个节点和 15 条依赖边。
- 系统不是单点功能，而是完整学习闭环。

### 第二步：动态学生画像

打开 `/profile`。

展示 7 个画像维度：知识基础、认知风格、学习能力、易错点、学习目标、兴趣方向、学习习惯。

重点讲：

- 画像来自自然语言对话和学习行为，不是静态表单。
- 每个维度有置信度。
- 画像会被资源生成、路径推荐和评估报告复用。

### 第三步：个性化资源生成

打开 `/resources`。

输入“神经网络”，选择文档、思维导图、练习题、代码、拓展阅读。

重点讲：

- 输出不只是一段文本，而是 5 类学习资源。
- Mermaid 思维导图本地渲染，现场网络不稳定也可展示。
- 代码案例可直接支撑实践学习。

### 第四步：画像驱动路径推荐

打开 `/path`。

这里上方是画像驱动推荐路径，下方是课程知识图谱。

重点讲：

- 推荐路径不是固定课程表。
- 系统读取画像、测验记录和薄弱点。
- 例如学生对计算机视觉感兴趣、神经网络已掌握，系统会把 CNN、CV 放入后续路径。

### 第五步：自适应学习闭环

打开 `/learning`。

选择一个知识点，系统会经历：

资料生成 -> 在线测试 -> 自动批改 -> 分析薄弱点 -> 生成补强资料 -> 再测。

重点讲：

- 这对应 A3 的“个性化资源生成”和“学习效果动态调整”。
- 系统不仅生成内容，还验证内容有没有被学生掌握。

### 第六步：评估报告

打开 `/evaluation`。

重点讲：

- 综合评分由画像质量、知识掌握、实践能力、自适应闭环、持续性共同计算。
- 页面展示 7/7 画像、13 份资源、5 类资源、2 次测验、91 分钟学习记录。
- 补强证据显示分数从 72 提升到 88。

## 5. 创新点 1 分钟

1. **对话式画像构建**：替代传统问卷，画像随学习过程动态变化。
2. **多智能体编排**：把生成、出题、答疑、评估拆成可解释的协作流程。
3. **资源生成和测评闭环结合**：不是只生成材料，而是生成、测试、补强、再测。
4. **画像驱动路径推荐**：路径推荐有可解释证据。
5. **答辩可验证证据**：驾驶舱、评估报告、学习时间线都能证明系统实际运行。

## 6. 可能追问和回答

### 问：你们和普通大模型聊天有什么区别？

答：普通聊天只有一次性回答。我们的系统有学生画像、知识图谱、多智能体协作、资源沉淀、测验记录和评估闭环。大模型只是生成引擎之一，真正的核心是个性化学习流程。

### 问：多智能体体现在哪里？

答：在对话和资源生成场景里，Planner 会识别任务，再调度 Profiler、Resource、Quiz、Tutor、Evaluator 等专职 Agent。前端对话页和驾驶舱都会展示智能体链路。

### 问：如何证明个性化？

答：个性化体现在三处：资源生成会读取画像，知识图谱推荐路径会读取画像和错因，评估报告会读取画像、测验和学习记录。演示数据中可以看到 7 维画像和个性化路径推荐。

### 问：如何减少大模型幻觉？

答：系统使用课程知识库和知识图谱约束生成内容；结构化内容经过 JSON 解析和兜底校验；演示数据和评估逻辑可在无模型调用时稳定展示关键闭环。

### 问：为什么选择人工智能导论？

答：人工智能导论覆盖机器学习、深度学习、NLP、CV、强化学习等知识点，层级清晰、先修关系明显，适合作为个性化学习路径和资源生成的示范课程。

## 7. 收尾 20 秒

总结来说，智学通完成了从“了解学生”到“生成资源”再到“评估补强”的完整闭环。我们的重点不是把大模型包装成一个问答窗口，而是把大模型放入可解释、可追踪、可评估的多智能体学习系统中，真正服务高等教育个性化学习。
