# 过拟合与正则化

## 1. 概述

过拟合指模型在训练集上表现很好，但在测试集上表现差。正则化是防止过拟合的主要手段。

## 2. 过拟合的表现

- 训练准确率高，测试准确率低
- 模型学习了数据中的噪声
- 模型复杂度过高

## 3. 常见原因

- 训练数据太少
- 模型参数太多（过于复杂）
- 训练时间过长
- 数据包含大量噪声

## 4. 解决方案

### L1/L2 正则化
在损失函数中加入参数惩罚项：
- L1: λΣ|wᵢ| → 产生稀疏解
- L2: λΣwᵢ² → 权重衰减

### Dropout
训练时随机丢弃部分神经元，防止共适应。

### 早停（Early Stopping）
当验证集误差不再下降时停止训练。

### 数据增强
通过旋转、翻转、裁剪等方式扩充训练集。

### 交叉验证
用K折交叉验证评估模型泛化能力。

## 5. 关键要点
1. 过拟合是机器学习最常见的问题
2. Dropout + L2正则化是深度学习标配
3. 增加训练数据是最好的防过拟合方法
4. 验证集用于调参，测试集用于最终评估
